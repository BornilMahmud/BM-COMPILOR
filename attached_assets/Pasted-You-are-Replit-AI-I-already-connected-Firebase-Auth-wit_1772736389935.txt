4) MANDATORY C + FLEX + BISON COMPILER CORE (compiler/bmcc)
========================
To satisfy “C + Flex + Bison” in this web project, bmcc must include a small compiler frontend built with Flex+Bison:
- Implement a tiny DSL called “BM Script” that describes a run job.

BM Script example (generated internally by bmcc or accepted as file):
  LANG c;
  FILE "main.c";
  RUN;

or
  LANG py;
  FILE "main.py";
  STDIN "hello\n";
  RUN;

Flex (bm_script_lexer.l):
- tokens: LANG, FILE, RUN, STDIN, STRING, IDENT, SEMI

Bison (bm_script_parser.y):
- grammar parses the script and builds a small AST/job struct.

bmcc must:
1) Build BM Script text from CLI args (language/file/stdin)
2) Parse it using Flex+Bison
3) If parse ok, execute the job:
   - For c: gcc file -o a.out then run
   - For cpp: g++ file -o a.out then run
   - For java: javac Out.java then java Out
   - For py: python3 file
4) Capture stdout/stderr, exit code
5) Output JSON when --json is passed.

IMPORTANT SAFETY IN bmcc:
- Use fork/execvp + pipes in runner.c
- Apply timeout (alarm or watchdog)
- Run inside the temp directory only
- No system()

Compiler/Makefile must build:
- flex -> build/lexer.c
- bison -> build/parser.c + build/parser.h
- gcc to produce ../bmcc
